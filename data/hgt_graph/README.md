# HGT heterogeneous graph dataset

Generated by:

```powershell
conda activate HGT
python scripts\preprocess\build_hgt_graph.py
```

The graph follows `docs/preprocess.md` at graph level. Metro stations are sample
anchors; nearby historic buildings, roads, POIs, conservation areas, and
administrative areas are typed nodes connected by typed spatial relations.

## Files

- `hgt_tensors.pt`: homogeneous HGT tensors: `x`, `node_type`,
  `edge_index`, `edge_type`, `edge_time`, type maps, and node offsets.
- `hetero_data.pt`: PyTorch Geometric `HeteroData` with typed node stores and
  typed edge stores.
- `pyhgt_graph.pkl`: legacy `pyHGT.data.Graph` object.
- `nodes/*.csv`: node metadata tables for inspection.
- `edges.csv`: edge table with source/target type, relation, ids, and weight.
- `manifest.json`: generation parameters and node/edge counts.

## Default graph scale

- Context radius: 2560 m around each station.
- Feature dimension: 16 for every node type.
- Node types: `station`, `building`, `road`, `poi`, `conservation_area`,
  `admin_area`.
- Relation types include station-near-building, station-near-road,
  station-near-poi by POI group, station-inside-area, and reverse relations.

## Smoke test

This command was used to verify that the full tensor graph runs through one HGT
layer on GPU:

```powershell
conda run -n HGT python -c "import torch, sys; sys.path.insert(0,'HGT'); from pyHGT.model import GNN; torch.set_grad_enabled(False); t=torch.load('data/hgt_graph/hgt_tensors.pt', map_location='cuda'); model=GNN(16,16,len(t['node_type_map']),len(t['edge_type_map']),2,1,dropout=0.0,use_RTE=False).cuda().eval(); y=model(t['x'],t['node_type'],t['edge_time'],t['edge_index'],t['edge_type']); torch.cuda.synchronize(); print(tuple(y.shape), y.device, torch.isfinite(y).all().item())"
```

Expected output:

```text
(286233, 16) cuda:0 True
```
